## Diagrama de DecisГөes Arquiteturais

```
Decision: SPA vs. SSR?
в”ңв”Җ SPA (Chosen)
в”Ӯ  в”ңв”Җ Pro: No server load, better UX, fast interactions
в”Ӯ  в”ңв”Җ Pro: Works offline, shareable URLs
в”Ӯ  в””в”Җ Con: Initial JS bundle (mitigated by code-splitting)
в”Ӯ
в””в”Җ SSR
   в”ңв”Җ Pro: Better SEO, faster first paint
   в””в”Җ Con: Server load, complex caching

Decision: CSV vs. Database?
в”ңв”Җ CSV (Chosen)
в”Ӯ  в”ңв”Җ Pro: Version control, easy to edit, no migration
в”Ӯ  в”ңв”Җ Pro: In-memory = fast, zero setup
в”Ӯ  в””в”Җ Con: Scales only to ~10k rows
в”Ӯ
в””в”Җ Database
   в”ңв”Җ Pro: Scales infinitely
   в””в”Җ Con: Extra complexity, need migrations

Decision: Zustand vs. Redux vs. Context?
в”ңв”Җ Zustand (Chosen)
в”Ӯ  в”ңв”Җ Pro: Minimal boilerplate, tiny bundle
в”Ӯ  в”ңв”Җ Pro: Per-module stores, no provider hell
в”Ӯ  в””в”Җ Con: Smaller ecosystem
в”Ӯ
в”ңв”Җ Redux
в”Ӯ  в”ңв”Җ Pro: Mature, time-travel debugging
в”Ӯ  в””в”Җ Con: Boilerplate-heavy for simple state
в”Ӯ
в””в”Җ Context
   в”ңв”Җ Pro: Built-in to React
   в””в”Җ Con: Causes unnecessary re-renders

Decision: Tailwind vs. CSS-in-JS vs. CSS Modules?
в”ңв”Җ Tailwind (Chosen)
в”Ӯ  в”ңв”Җ Pro: Rapid development, consistency
в”Ӯ  в”ңв”Җ Pro: Small final bundle, purging support
в”Ӯ  в””в”Җ Con: Learning curve for developers new to utility-first
в”Ӯ
в””в”Җ Others
   в””в”Җ Con: More complex, more JS in bundle
```
